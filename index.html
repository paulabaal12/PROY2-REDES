Proyecto

Resumen del Proyecto

Diseño e implementación de una infraestructura de red en AWS para una empresa de desarrollo inmobiliario en proceso de construcción de 2 edificios de apartamentos y un condominio de casas.

Objetivos Principales

  • Red privada segmentada para 15 Arquitectos
  • Red de administración para 3 miembros del equipo de TI
  • Red para 10 Ingenieros de obra
  • Servidores de renderización y almacenamiento de planos
  • Página web pública con información de proyectos
  • Servicios internos: DHCP, NTP, LDAP, DNS
  • Implementación de medidas de seguridad avanzadas
  • VPN para acceso remoto seguro

Seguridad

Control de Acceso y Seguridad

Políticas de Firewall

  • TI posee acceso administrativo completo a todas las subredes
  • Arquitectos pueden acceder a Render, Storage, Intranet e Internet
  • Ingenieros tienen acceso restringido únicamente a Storage (solo lectura)
  • DMZ completamente aislada, accesible exclusivamente desde Internet público
  • NAT Gateway proporciona salida controlada a Internet para subredes privadas
  • SSH permitido solamente desde Security Group de TI y Bastion Host
  • Hardening aplicado: todos los puertos no utilizados están bloqueados
  • Network ACLs configuradas para doble capa de seguridad

Aplicaciones

Casos de Uso Empresariales

Colaboración en Diseño Arquitectónico

Los 15 arquitectos trabajan simultáneamente en proyectos BIM almacenados en EFS. Acceden de forma segura mediante VPN desde cualquier ubicación, permitiendo trabajo remoto sin comprometer la seguridad de los proyectos confidenciales.

  • Acceso concurrente a planos 3D mediante EFS con baja latencia
  • Control de versiones automático para prevenir pérdidas de trabajo
  • Renderizado en servidores dedicados liberando recursos locales
  • Sincronización en tiempo real entre equipos distribuidos

Supervisión de Obra por Ingenieros

Los 10 ingenieros consultan planos actualizados desde obra mediante dispositivos móviles conectados a VPN. El acceso es estrictamente de solo lectura, garantizado por Security Groups y NACLs.

  • Visualización de planos siempre actualizados desde campo
  • Reducción de errores constructivos por uso de versiones obsoletas
  • Acceso controlado mediante autenticación LDAP corporativa
  • Logs de auditoría de todos los accesos a documentos

Gestión Centralizada por TI

El equipo de 3 técnicos administra toda la infraestructura mediante CloudWatch, Systems Manager y acceso SSH privilegiado. Monitorean rendimiento, gestionan usuarios en LDAP y mantienen la disponibilidad del sistema.

  • Dashboard unificado de monitoreo con CloudWatch
  • Administración remota sin bastion mediante Systems Manager
  • Gestión centralizada de usuarios y permisos con LDAP
  • Alertas automáticas ante incidentes de seguridad o rendimiento

Promoción Digital de Proyectos

El sitio web público en la DMZ muestra proyectos inmobiliarios con renders de alta calidad. Clientes potenciales pueden explorar propiedades, solicitar información y agendar visitas mediante formularios seguros.

  • Disponibilidad 24/7 con alta disponibilidad mediante ALB
  • Certificado SSL para transmisión segura de datos de clientes
  • CDN para entrega rápida de imágenes y renders pesados
  • Aislamiento completo de la red interna mediante DMZ

Cumplimiento Normativo y Auditoría

La arquitectura implementa controles de seguridad alineados con estándares internacionales para protección de datos empresariales y confidencialidad de proyectos.

  • Encriptación en tránsito y en reposo para todos los datos
  • Logs de auditoría inmutables con CloudTrail
  • Backup automático y disaster recovery probados
  • Segregación de funciones y principio de mínimo privilegio
